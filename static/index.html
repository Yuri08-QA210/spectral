ACCESS RESTRICTION

The keystream oracle only serves bytes from early stream positions. The encrypted target is located at a significantly later offset — direct retrieval is unavailable.

API Reference
GET
/api/info
Challenge parameters and runtime configuration
GET
/api/flag
Encrypted flag (hex-encoded) and its stream offset
GET
/api/keystream?start=<int>&length=<int>
Request keystream bytes from a given position
start: byte offset (0-indexed) length: max MAX_KEYSTREAM_LENGTH start < MAX_KEYSTREAM_START
POST
/api/encrypt
Encrypt arbitrary plaintext — keystream starts from position 0
body: {"plaintext": "<hex>"} max 64 bytes
